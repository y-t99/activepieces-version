Properties are used to collect from the user information to execute an action or an trigger, a property could be `short text`, `long text`, `checkbox`, `number`, `dropdown`, or `auth2`.

All properties `name` should be a snake case.

# Schema

Here is the schema of each property type.

## Short text

```typescript
Property.ShortText({
	displayName: string;
	description: string | undefined;
	required: boolean;
})
```


## Long text

```typescript
Property.LongText({
	displayName: string;
	description: string | undefined;
	required: boolean;
})
```

## Checkbox 

```typescript
Property.Checkbox({
	displayName: string;
	description: string | undefined;
	required: boolean;
})
```


## OAuth2 

```typescript
Property.Number({
    displayName: string,
    required: boolean,
    authUrl: string, // The url used for redirection
    tokenUrl: string, // The url used for claiming token
    scope: string[]],
})
```


## Number 

```typescript
Property.Number({
	displayName: string;
	description: string | undefined;
	required: boolean;
})
```

## Dropdown 

Options is function, so you can load options dynmiucally depends on the user props value.

```typescript
Property.Dropdown({
    displayName: string;
    description: string | undefined;
    required: boolean;
    options: async (propsValue): DropdownState {
        if (props['authentication'] === undefined) {
            return {
                disabled: true;
            }
        }
        return {
            options: [{
                label: "Option One",
                value: "1"
            }]
        }
    }
})
```
